Testcase: AES_DECRYPT

Run the following command from "src" folder:
There are 4 top level verilogs hence we have to run parser 4 times.

1.python parser.py -x -f benchmark_examples/CHStone/aes_decrypt/console_input/aes_decrypt_call_input.txt -t aes_decrypt_call benchmark_examples/CHStone/aes_decrypt/verilog/aes_decrypt_call_new.v benchmark_examples/CHStone/aes_decrypt/verilog/aes_decrypt_call_dEe.v benchmark_examples/CHStone/aes_decrypt/verilog/aes_decrypt_call_eOg.v
2.python parser.py -f benchmark_examples/CHStone/aes_decrypt/console_input/InversShiftRow_ByteS_input.txt -t InversShiftRow_ByteS benchmark_examples/CHStone/aes_decrypt/verilog/InversShiftRow_ByteS_new.v benchmark_examples/CHStone/aes_decrypt/verilog/InversShiftRow_Bybkb.v
3.python parser.py -f benchmark_examples/CHStone/aes_decrypt/console_input/AddRoundKey_input.txt -t AddRoundKey benchmark_examples/CHStone/aes_decrypt/verilog/AddRoundKey_new.v
4.python parser.py -f benchmark_examples/CHStone/aes_decrypt/console_input/AddRoundKey_InversMi_input.txt -t AddRoundKey_InversMi benchmark_examples/CHStone/aes_decrypt/verilog/AddRoundKey_InversMi_new.v benchmark_examples/CHStone/aes_decrypt/verilog/AddRoundKey_Invercud.v

Note: To input the RAM, ROM and Function Input from console remove -f and the text file path from above command.
